Simplified Index Management

Note: this feature is currently in Beta mode.

Simplified Index Management for AEM as a Cloud Service allows to customize indexes by specifying just the differences (called "diff.index") to the out-of-the-box (OOTB) indexes.

There is no need to copy definitions, or to explicitly define versions. When deploying the diff, the changes are automatically merged with the latest out-of-the-box index, and if needed, a new version is automatically created.

These indexes have a property "mergeInfo" that contains a link to this documentation.

For more information about indexing, see: Content Search and Indexing.

A "diff.index" is a special node under /oak:index named /oak:index/diff.index that carries a JSON "diff" describing additions to existing OOTB index definitions. The diff content is stored as a JSON file under diff.json/jcr:content/jcr:data, and maps index names to partial fragments. Fragments only contain the properties you want to add to the index. During pipeline run, if needed a new version of an index (e.g., damAssetLucene-12-custom-4) is created by applying the additions on top of the latest product index (e.g., damAssetLucene-12). Existing fields and property values in the OOTB index are preserved; only new properties are added.
